# HR MVP 범위 정의

## 1. 문서 역할

이 문서는 HR 입퇴사/조직도 조회 MVP의 `제품 범위(product scope)`를 정의한다.

아래 내용은 이 문서가 기준이다.

- 누구를 위한 MVP인가
- 무엇을 조회해야 하는가
- 1차 MVP에 무엇을 포함하고 제외할 것인가
- 어떤 구현 방향이 현실적인가

데이터 규칙과 계산 로직의 기준 문서는
[docs/operations/data-rules.md](/Users/shlee/Developments/hr-side/docs/operations/data-rules.md)다.

## 2. 목표

직원 100명 내외 규모의 회사에서 2015년부터 현재까지의 인사 이력을 바탕으로,
특정 날짜 기준의 조직도와 주요 HR 통계를 조회할 수 있는 MVP를 만든다.

핵심 질문은 아래 3가지다.

1. 특정 날짜 `D`에 누가 재직 중이었는가
2. 그 시점에 각 직원은 어느 팀에 속해 있었는가
3. 그 날짜가 속한 연도의 입사/퇴사/휴직 통계는 무엇인가

## 3. 사용자 시나리오

사용자는 로컬 저장소의 CSV 파일을 직접 수정한다.
그 다음 로컬 명령으로 HTML 리포트를 생성하고 브라우저에서 기준 날짜를 바꿔가며 조회한다.

조회 결과로 아래 정보를 본다.

- 해당 날짜 기준 조직도
- 팀별 재직 인원
- 팀별 근무 인원
- 팀별 휴직 인원
- 기준 연도 입사자 수
- 기준 연도 퇴사자 수
- 기준 연도 순증감

## 4. 1차 구현 방향

### 4.1 권장안

`로컬 CSV + 단일 HTML 리포트 생성기`로 시작한다.

이유는 아래와 같다.

- 별도 SaaS 계정, 권한, Apps Script 배포가 필요 없다
- `git clone` 후 각 로컬에서 독립적으로 실험할 수 있다
- CSV diff가 남아서 변경 추적이 쉽다
- 정적 HTML이라 공유와 보관이 쉽다

### 4.2 웹 전환 조건

아래 조건이 생기면 웹 앱 전환을 고려한다.

- 조회 사용자가 많아짐
- 접근 권한 분리가 필요함
- 시각화 요구사항이 커짐
- 외부 시스템 연동이 필요함

## 5. 1차 MVP 포함 범위

### 5.1 필수 입력

- 로컬 CSV 파일 세트
- 기준 날짜 1개

### 5.2 필수 출력

- 기준 날짜
- 총 재직 인원
- 총 근무 인원
- 총 휴직 인원
- 기준 연도 입사자 수
- 기준 연도 퇴사자 수
- 팀별 인원 집계
- 조직도 트리

### 5.3 1차 조직도 표현 방식

처음부터 복잡한 박스형 chart보다 아래 방식 중 하나를 쓴다.

1. 들여쓰기 기반 트리 표
2. 팀별 테이블 + 상위팀 컬럼
3. HTML Sidebar 기반 간단한 org chart

1차 권장안은 `들여쓰기 기반 트리 표`다.

## 6. 1차 MVP 제외 범위

- 결재 워크플로우
- 인사평가
- 급여 연동
- 상세 직급 체계 관리
- 다중 법인 / 다중 국가 처리
- 고급 권한 관리
- 실시간 동시편집 충돌 제어

## 7. 성공 기준

MVP는 아래 조건을 만족하면 성공으로 본다.

1. 특정 날짜 기준 재직자 수를 일관되게 계산할 수 있다
2. 특정 날짜 기준 조직도를 복원할 수 있다
3. 기준 연도 퇴사자 수를 사람 손계산과 맞출 수 있다
4. HR 담당자가 스프레드시트만으로 데이터를 유지할 수 있다

## 8. 로드맵

### Step 1. 구조 확정

- 데이터 모델 확정
- 계산 규칙 확정
- 시트 컬럼 확정

### Step 2. 샘플 데이터 준비

- 실제 형태와 유사한 샘플 데이터 입력
- 예외 케이스 포함

### Step 3. 로컬 리포트 MVP

- CSV 로드
- 기준일 스냅샷 계산
- 조직도 트리 생성
- KPI 계산

### Step 4. 검증

- 수기 검증용 날짜 3~5개 선정
- HR 담당자와 숫자 검증
